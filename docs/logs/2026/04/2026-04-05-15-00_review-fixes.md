# Review fixes

Implemented the requested review-fix pass focused on the concrete issues.

## Fixed

- Removed duplicate `CheckpointManager` construction path by stopping `StageBuilder` from creating one and passing the runner-owned checkpoint manager through `StageContext`.
- Resume logic now consults SQLite checkpoint state before skipping a stage; only `completed` stages are resumed.
- `CheckpointManager.load_stage()` now returns stage metadata including status/counts, not just a path.
- Language filter now emits a warning when configured with a non-English allowed set that currently has no heuristic support.
- Seed source now raises a clear error when `data_source.path` is missing for `type='seeds'`.
- Labeling filter now wraps missing rubric files with a config-aware error mentioning `filters.labeling_engine.rubric_path`.
- Canary handling now uses explicit rubric schema labels via `few_shot[].expected_verdict: pass|fail`.
- Generator checkpoint `response_count` is updated after each raw response write, avoiding stale counts after mid-loop failure.
- Diversity scoring now delegates through `Record.text_for_diversity()` instead of reaching into `ConversationRecord` payload internals.
- Source stage method signatures now accept `list[Record]`, avoiding Liskov-unsafe narrowing.
- `fastembed` import in runner is now lazy inside the HuggingFace embedding path.
