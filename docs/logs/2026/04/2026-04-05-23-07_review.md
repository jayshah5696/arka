# Embeddings config update

Adjusted diversity embedding configuration per request.

## What changed
- Default local embeddings now use HuggingFace-style model naming via FastEmbed
- Default model is locked to `all-MiniLM-L6-v2`
- User can override the embedding model path/name in config
- Two embedding backends are supported for diversity scoring:
  - `huggingface` — local via FastEmbed, defaulting to `sentence-transformers/all-MiniLM-L6-v2`
  - `openai` — provider/OpenAI-compatible embedding APIs, including OpenRouter-compatible endpoints

## Config shape
```yaml
embeddings:
  provider: huggingface   # or openai
  model: all-MiniLM-L6-v2
```

OpenAI-compatible example:
```yaml
embeddings:
  provider: openai
  model: text-embedding-3-small
  api_key: ${OPENROUTER_API_KEY}
  base_url: https://openrouter.ai/api/v1
  openai_compatible:
    referer: https://example.com
    title: arka
```

## Behavior
- For `huggingface`, bare names like `all-MiniLM-L6-v2` are normalized to `sentence-transformers/all-MiniLM-L6-v2`
- Full model paths containing `/` are passed through unchanged
- Diversity score continues to return `null` if embedding generation fails
