# Generator issue fixes

Implemented and verified the requested generator hardening changes.

## Changes made

- Fixed structured-generation branching in `src/arka/pipeline/generator_stages.py`
  - `PromptBasedGeneratorStage._complete_raw()` now calls `complete_structured()` instead of incorrectly preferring `complete()`.
  - It forwards `temperature` and `max_tokens` when supported, with a safe fallback for simple test doubles that do not accept those kwargs.
  - Added `_generated_text_from_output()` so resume/raw-response persistence still works even when a provider returns `parsed` without `text`.

- Removed brittle checkpoint root inference
  - `PromptBasedGeneratorStage` now accepts `project_root` in its constructor.
  - `StageBuilder` now injects a shared `CheckpointManager(project_root / "state.db")` and `project_root` into the generator stage.
  - Generator fallback root discovery no longer relies on `ctx.work_dir.parents[3]`; it searches for the `runs/` ancestor and errors clearly if not found.

- Hardened parse failure handling
  - `_parse_responses()` now skips malformed generator outputs instead of raising on the first bad row.
  - Parse failures are logged as warnings.
  - Dropped rows are written to `dropped.parquet` with `StageEvent(reason_code="generator_parse_failure")`.
  - Stage stats are written with `dropped_count` and `drop_reasons`, so run reports can surface the loss.
  - The previous fatal count-mismatch exception was removed.

- Cleaned export surface
  - Added `__all__ = ["PromptBasedGeneratorStage", "compute_prompt_hash"]` to `generator_stages.py`.
  - Replaced wildcard re-export in `generation_stages.py` with explicit named exports.

- Extended `LLMClient.complete_structured()`
  - Added optional `temperature` and `max_tokens` support and threaded those kwargs through all structured-output strategies.

## Tests added/updated

- Generator tests now cover:
  - structured path usage
  - partial raw-response resume
  - malformed response skipping with warning/stats/dropped parquet
  - injected checkpoint manager usage
  - compatibility shim export surface

- Stage builder tests now assert generator receives injected `project_root` and checkpoint manager.
- LLM client tests now assert `complete_structured()` forwards `temperature` and `max_tokens` to native parse.
