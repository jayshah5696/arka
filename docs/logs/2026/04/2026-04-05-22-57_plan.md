# SFT completeness gaps implementation

Implemented the requested Tier A completeness work from `docs/PLAN.md` / aligned `docs/SPEC.md` areas for the current SFT slice.

## What changed

### 1. Near-dedup stage
- Added `dedup.near` config with:
  - `enabled`
  - `shingle_size`
  - `num_hashes`
  - `jaccard_threshold`
- Added `NearDedupStage` in `src/arka/pipeline/dedup_stages.py`
- Wired it into `StageBuilder` after exact dedup
- Current implementation uses token 5-gram MinHash-style signatures and drops:
  - `near_duplicate_minhash`
- Writes:
  - `dropped.parquet`
  - `clusters.parquet`
  - `stats.json`

### 2. Dataset report completions
- Runner now writes:
  - `runs/<run_id>/report/samples.jsonl`
  - `runs/<run_id>/report/canaries.json`
- `run_report.json` now includes:
  - `samples_path`
  - `canaries_path`
  - `canaries`
  - `diversity_score`
- `samples.jsonl` contains up to 20 deterministic random examples
- `canaries.json` is derived from rubric few-shot examples when labeling is enabled; otherwise:
  - `known_good: []`
  - `known_bad: []`
  - `status: null`
- Added diversity-score computation using provider embeddings (`text-embedding-3-small`) plus in-process k-means/entropy calculation
  - returns `null` if embeddings are unavailable/fail

### 3. Output formatters
- `output.format` is now honored in `OutputWriter`
- Supported formats:
  - `jsonl`
  - `chatml`
  - `alpaca`
- Final dataset and `samples.jsonl` both use the selected formatter

## Config/docs updates
- Added `dedup.near.enabled: false` to baseline configs
- Switched smoke/example outputs to `chatml` where tests now validate formatter behavior
- Updated README implemented-path summary and artifact list

## Notes / limitations
- Near dedup currently implements the MinHash path only; the earlier SimHash note in `SPEC.md` is not separately wired yet.
- Diversity score is best-effort and returns `null` when embedding calls are unavailable; no fake pricing table or local embedding fallback was introduced.
- `canaries.json` uses rubric few-shot examples, matching the decision to avoid a new canary config surface for now.
