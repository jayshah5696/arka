## Changes made

### LLM / structured output
- Deduplicated OpenAI client construction into `src/arka/llm/openai_client.py` via `build_openai_client()`.
- `src/arka/llm/client.py` now uses that shared constructor.
- `complete_structured()` now tries provider-native structured parsing first (`beta.chat.completions.parse(..., response_format=PydanticModel)`) and falls back to prompt+parse only if native parse is rejected.
- Replaced fixed retry wait with exponential backoff.
- Added coded `LLMClientError` values (`auth_error`, `bad_request_error`, `retryable_api_error`, `parse_error`, `invalid_structured_response`).

### Runtime safety / batching
- Added `src/arka/common/concurrency.py` with shared bounded worker-count logic.
- `LLMClient.complete_batch()` now caps default worker count.
- `LabelingEngine.label_batch()` now uses the same bounded worker count.

### Pipeline failure handling
- `PipelineRunner` now records failed runs explicitly instead of leaving them as `running`.
- On stage failure it now writes failed `manifest.json` and `run_report.json`, includes stage-level error metadata, and marks SQLite run status as `failed`.
- Added `StageErrorInfo` and stage-stat error serialization.

### Seed identity / hashes
- `SeedSourceStage` now uses content-stable SHA-256 record ids instead of `seed-{row_index}`.
- Seed `content_hash`, `seed_file_hash`, and runner `config_hash` now use full SHA-256 hex digests.
- Rubric hash and judge prompt hash also now use full SHA-256 hex digests for consistency.

### Label-path failure taxonomy
- `LabelingQualityFilterStage` now classifies label failures into dropped-record reasons:
  - `label_parse_failure`
  - `label_auth_failure`
  - `label_retryable_api_failure`
  - `invalid_structured_response`
- Those failures are persisted to `dropped.parquet` and surfaced in `stats.json` like other drop reasons.
